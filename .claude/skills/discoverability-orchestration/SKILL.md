# Discoverability Orchestration

Use this skill whenever BayShine's discoverability layer needs work. Discoverability covers SEO, GEO, AI search, schema markup, site architecture, internal linking, and citation-readiness.

---

## Which specialist handles what

Use this decision tree to route work to the right subagent.

| Situation | Specialist |
|---|---|
| Need schema for a new BayShine page | Schema Specialist |
| Existing schema is broken, outdated, or missing | Schema Specialist |
| Planning structure for a new site section | Site Architecture Specialist |
| Content organization feels unclear or competing | Site Architecture Specialist |
| BayShine not appearing in ChatGPT or Perplexity for local queries | GEO/AI Search Specialist |
| Unsure if AI crawlers can access the site | GEO/AI Search Specialist |
| Need to create or update llms.txt | GEO/AI Search Specialist |
| Pages are not getting enough internal links | Internal Linking Specialist |
| Anchor text patterns look repetitive or spammy | Internal Linking Specialist |
| Content ranks but is not being cited by AI engines | Citation-Readiness Specialist |
| Blog posts are too long or unstructured for AI to extract answers | Citation-Readiness Specialist |
| New blog post about to publish and needs structure review | Citation-Readiness Specialist |

---

## How to invoke a specialist

Within Claude Code, address the specialist directly by name in your prompt:

```
As the Schema Specialist, audit the BayShine homepage and recommend 
schema additions or corrections.
```

```
As the Internal Linking Specialist, identify which blog posts have 
zero inbound internal links and recommend the three highest-impact 
links to add first.
```

```
As the GEO/AI Search Specialist, audit robots.txt for AI crawler 
access and produce a draft llms.txt for bayshine.net.
```

```
As the Citation-Readiness Specialist, audit the ceramic-coating 
service page and the three most recent blog posts. Rank which page 
needs restructuring most urgently.
```

You can also invoke two specialists together when the task spans both scopes:

```
As the Site Architecture Specialist and Internal Linking Specialist 
working together, propose how to add a service area page for ZIP 
34655 (Trinity, FL) including URL placement, navigation entry, and 
inbound link sources from existing content.
```

## Multi-specialist workflows

Some tasks require multiple specialists in sequence. Use these patterns.

### Full discoverability audit

All five specialists audit independently and produce parallel reports. The operator reviews all five before any remediation begins.

1. Schema Specialist: schema audit across all page types
2. Site Architecture Specialist: architecture and URL structure audit
3. GEO/AI Search Specialist: AI crawler access, entity markup, citation-readiness assessment
4. Internal Linking Specialist: link graph audit, orphan page identification
5. Citation-Readiness Specialist: content structure audit on highest-priority pages

Each produces a report saved to `discoverability/`. Operator reviews, then specialists collaborate on a unified remediation priority list.

### New service page launch

Sequence before any new service page goes live:

1. Site Architecture Specialist confirms URL pattern and placement in hierarchy
2. Schema Specialist generates Service and LocalBusiness schema for the page
3. Internal Linking Specialist identifies which existing pages should link to the new page at launch
4. Citation-Readiness Specialist reviews the page structure for passage-level quotability

### New service area page launch

Sequence before any new zip-code or neighborhood page goes live:

1. Site Architecture Specialist confirms URL pattern, hierarchy placement, and content depth requirements
2. Schema Specialist adds geographic schema (areaServed, GeoCircle if appropriate)
3. Internal Linking Specialist plans cross-links to relevant service pages and from the service area hub
4. Citation-Readiness Specialist verifies the page has quotable geographic-specific content

### Blog post launch (automated pipeline)

For each post generated by the blog automation:

1. Citation-Readiness Specialist sets structural standards (these are embedded in the blog prompt, but audit weekly posts manually to catch drift)
2. Schema Specialist confirms Article schema is correct in the blog post template
3. Internal Linking Specialist verifies the new post has inbound links from at least two existing relevant pages

### AI search optimization push

When BayShine is not appearing in AI-generated local recommendations:

1. GEO/AI Search Specialist audits AI crawler access and content visibility
2. Citation-Readiness Specialist restructures the highest-priority pages for passage-level retrieval
3. Schema Specialist adds or corrects entity markup
4. Internal Linking Specialist routes link equity toward the pages being optimized

## Output collation

Save all specialist outputs to the `discoverability/` directory at the BayShine repo root.

File naming convention: `[YYYY-MM-DD]-[specialist-name]-[scope].md`

Examples:
- `discoverability/2026-05-07-schema-homepage-audit.md`
- `discoverability/2026-05-07-internal-linking-full-audit.md`
- `discoverability/2026-05-07-geo-ai-robots-llmstxt.md`

Create the `discoverability/` directory if it does not exist.
