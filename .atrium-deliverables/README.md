# Atrium deliverables — courier directory

These are Atrium products, written in this Claude Code session, riding
along on BayShine's authorized git push channel because Atrium does not
yet have its own GitHub remote. Once Atrium is pushed to
`github.com/constantine-png/Atrium`, this directory is removed in a
follow-up commit and the deliverables move to their canonical home at
`Atrium/library/artifacts/`.

## How to ship each

Each directory contains a `PUBLISH-NOTES.md` (or equivalent) with the
exact next steps. Read the notes first, then take the action.

The Atrium voice rules from `Atrium/spine/origins/CLAUDE.md` apply to
every published artifact. Do not add em-dashes, exclamation marks, or
soft marketing voice when copying these into the destination platforms.
The voice is already correct here; preserve it.

## Why this directory exists at all

In an ideal sequence, Atrium would be its own GitHub repo with its own
authorized push, and these artifacts would be authored directly into
`library/artifacts/`. The Claude Code session that wrote them was
scoped to BayShine for git push. Rather than block on the scope
expansion, the deliverables were authored here and will be migrated
once the operator (Constantine) connects Atrium to a computer with
push permissions for `constantine-png/Atrium`.

Pragmatism over elegance, per the operator's standing instruction.
